# 📸 图片使用指南

## 🎯 快速开始

### 1. 设置头像

1. **准备头像图片**
   - 推荐尺寸：200x200 或 300x300 像素
   - 格式：JPG 或 PNG
   - 命名：`avatar.jpg` 或 `avatar.png`

2. **放置图片**
   ```
   将头像放到：assets/images/avatar.jpg
   ```

3. **配置已自动完成**
   - `_config.yml` 中已配置：`avatar: /assets/images/avatar.jpg`
   - 首页会自动显示头像

### 2. 在文章中使用图片

#### 步骤：
1. 将图片放到 `assets/images/posts/` 目录（可以按年份分类）
2. 在文章中使用 Markdown 语法引用

#### 示例文章：

```markdown
---
layout: post
title: "我的技术文章"
date: 2025-01-16 10:00:00 +0800
categories: 技术 前端
---

# 文章标题

这是一段文字说明。

## 插入图片示例

![图片描述](/assets/images/posts/2025/example-image.jpg)

图片会自动适应宽度。

## 控制图片大小

使用 HTML 标签可以控制图片大小：

<img src="/assets/images/posts/2025/example-image.jpg" alt="图片描述" width="600">

或者使用样式：

<img src="/assets/images/posts/2025/example-image.jpg" alt="图片描述" style="max-width: 100%; height: auto; border: 1px solid #ddd; padding: 5px;">
```

## 🎨 图片优化建议

### 格式选择
- **JPG**：适合照片、截图（文件小）
- **PNG**：适合图标、需要透明背景的图片
- **WebP**：现代格式，压缩率高（浏览器需支持）

### 压缩工具
- [TinyPNG](https://tinypng.com/) - 在线压缩，支持 JPG 和 PNG
- [Squoosh](https://squoosh.app/) - Google 的工具，支持多种格式
- [ImageOptim](https://imageoptim.com/) - Mac 客户端

### 大小建议
- 单张图片建议不超过 1MB
- 对于长文章，建议压缩图片以提高加载速度

## 📝 Markdown 图片语法

### 基本语法
```markdown

```

### 示例
```markdown
![代码截图](/assets/images/posts/2025/code-screenshot.png "代码示例")

![项目演示](/assets/images/posts/2025/project-demo.jpg)
```

## 🔗 使用外部图片

如果图片托管在其他地方（如 GitHub、图床），也可以直接使用链接：

```markdown
![图片](https://example.com/image.jpg)
```

**注意**：使用外部链接时，确保链接稳定，避免图片失效。

## 💡 小技巧

1. **图片命名**：使用有意义的文件名，如 `jekyll-theme-setup.png` 而不是 `img1.jpg`
2. **图片组织**：按年份或文章分类组织图片，便于管理
3. **图片说明**：为图片添加描述文字，提升可读性和 SEO
4. **响应式**：Cayman 主题会自动让图片适应移动端
